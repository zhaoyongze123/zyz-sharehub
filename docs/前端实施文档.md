适用范围：

- 首页与登录页视觉保真实现
- 资料分享、学习路线、笔记、简历工作台、个人中心、后台治理
- GitHub OAuth 登录接入
- 与 Spring Boot 后端接口联调

当前模式：

- 严格模式
- 设计先行
- 前端先完成结构与交互，再分阶段接入真实接口

## 2. 产品定位

`zyz-sharehub` 不是普通博客，而是面向学生与初级求职者的知识资料分享平台。

核心价值：

- 分享学习资料
- 发布学习路线
- 整理个人笔记
- 生成与导出简历
- 通过社区互动沉淀内容

产品结构分三层：

1. 公共内容层：浏览、搜索、发现优质资料和路线
2. 创作工作台：发布资料、写笔记、编辑路线、生成简历
3. 后台治理层：审核、举报、用户和内容管理

## 3. 技术方案

### 3.1 核心技术栈

- 框架：`Vue 3`
- 构建：`Vite`
- 语言：`TypeScript`
- 路由：`Vue Router`
- 状态管理：`Pinia`
- 请求层：`Axios`
- 样式：`SCSS + Design Tokens`
- 工具类样式：`UnoCSS`
- 富文本方案：`Markdown 编辑器 + 实时预览`
- 图表：`ECharts`

### 3.2 选型原则

- 视觉层不依赖现成后台模板
- 页面风格统一继承首页视觉语言
- 组件优先，页面做组合
- 所有颜色、字号、间距、圆角、阴影统一走 Token
- 先静态高保真，再接口联调

### 3.3 运行形态

- 前端采用 `SPA`
- 通过 `Nginx` 托管构建产物
- 登录由后端 `GitHub OAuth` 发起和回调
- 前端通过用户信息接口确认登录状态

## 7. 路由设计

### 7.1 路由分层

- 公共路由：任何人可访问
- 用户路由：登录后访问
- 管理路由：管理员访问

### 7.2 路由表

```ts
const routes = [
  { path: '/', name: 'home' },
  { path: '/login', name: 'login' },
  { path: '/resources', name: 'resources' },
  { path: '/resources/:id', name: 'resource-detail' },
  { path: '/roadmaps', name: 'roadmaps' },
  { path: '/roadmaps/:id', name: 'roadmap-detail' },
  { path: '/notes', name: 'notes' },
  { path: '/notes/:id', name: 'note-detail' },
  { path: '/publish/resource', name: 'publish-resource', meta: { auth: true } },
  { path: '/publish/roadmap', name: 'publish-roadmap', meta: { auth: true } },
  { path: '/editor/note/:id?', name: 'note-editor', meta: { auth: true } },
  { path: '/resume', name: 'resume', meta: { auth: true } },
  { path: '/me', name: 'me', meta: { auth: true } },
  { path: '/admin', name: 'admin', meta: { auth: true, admin: true } }
]
```

### 7.3 路由守卫

- 未登录访问受限页面时跳转登录页
- 登录后返回原目标页
- 非管理员进入后台时跳转 `403`
- 页面切换时统一处理标题、滚动位置和埋点

### 11.3 错误处理策略

- 403：跳权限不足页
- 404：显示资源不存在
- 500：显示服务异常并允许重试
- 上传失败：保留表单状态并提示重试
